# scam-survey — フィッシング/詐欺メッセージ受容実験サイト

日本語L1話者を対象に、詐欺的なセキュリティ警告メッセージが
①言語（日本語／英語）②フレーミング（positive／negative）
によってどう受け取られるかを調べる実験用サイトです。

ビルド不要の静的サイト（HTML/CSS/JS のみ）なので、
GitHub Desktop でリポジトリに追加 → push → Vercel でインポート、
という流れでそのまま公開できます。

---

## 1. ファイル構成

```
scam-survey/
├── index.html          … 同意画面 → 模擬警告モーダル＋アンケート → 送信完了画面
├── style.css            … デザイン（Googleフォーム風のシンプルなUI）
├── app.js                … 画面遷移・カウントダウン・データ収集ロジック
├── config.js             … グループごとの文言、GASの送信先URLなどの設定
├── apps-script/
│   └── Code.gs           … Googleスプレッドシートにデータを書き込むバックエンド
└── README.md              … このファイル
```

---

## 2. データ収集先（Googleスプレッドシート）を用意する

Vercelは静的ファイルしかホストできないため、回答データの保存には
**Google Apps Script（GAS）をバックエンド代わりに使う方法**を採用しています。
Googleフォームと同じ感覚で、回答が自動的にスプレッドシートの行として溜まっていきます。

1. Google スプレッドシートを新規作成する（例：「詐欺メッセージ実験_回答」）。
2. メニューの **拡張機能 → Apps Script** を開く。
3. デフォルトで開かれている `Code.gs` の中身を全部消し、
   このリポジトリの `apps-script/Code.gs` の内容を貼り付けて保存する。
4. 右上の **デプロイ → 新しいデプロイ** をクリック。
   - 種類の選択（歯車アイコン）で **ウェブアプリ** を選ぶ。
   - 「次のユーザーとして実行」→ **自分**
   - 「アクセスできるユーザー」→ **全員**（これを選ばないと外部からのPOSTが拒否されます）
   - **デプロイ** をクリックし、Google アカウントの承認を行う。
5. 発行された **ウェブアプリのURL**（`https://script.google.com/macros/s/.../exec` の形式）をコピーする。
6. ブラウザでそのURLに直接アクセスし、「OK: このエンドポイントは稼働しています…」と表示されれば成功。

> 一度デプロイした後に `Code.gs` を修正した場合は、
> 「デプロイを管理 → 編集 → バージョン：新しいバージョン → デプロイ」で更新してください
> （URLは変わりません）。

---

## 3. `config.js` を編集する

`config.js` を開き、5. でコピーしたURLを貼り付けます。

```js
const GAS_ENDPOINT = "https://script.google.com/macros/s/XXXXXXXXXXXX/exec";
```

空のままでも動作確認はできますが、その場合データはどこにも保存されず、
ブラウザのコンソールに出力されるだけになります（テスト用）。

同じファイルの `COUNTDOWN_SECONDS`（モーダルのカウントダウン秒数）や、
`GROUPS` 内の各グループの文言もここで調整できます。

`index.html` 内の同意文にある
`［担当者名を記入］` `［メールアドレスを記入］` の部分は、
実際の研究者名・連絡先に書き換えてください。

---

## 4. GitHub Desktop → Vercel で公開する

1. GitHub Desktop で新しいリポジトリを作成し（または既存のものを開き）、
   この `scam-survey` フォルダの中身（`index.html`・`style.css`・`app.js`・
   `config.js`・`apps-script/`・`README.md`）をリポジトリのルートにコピーする。
2. GitHub Desktop 上で変更内容を確認し、コミットメッセージを入力して
   **Commit to main**。
3. 右上の **Push origin** でGitHubにpushする。
4. [vercel.com](https://vercel.com) にログインし、**Add New → Project** から
   このGitHubリポジトリをインポートする。
5. Framework Preset は **Other**（静的サイト）のままでOK。
   Build Command / Output Directory は空欄のままで問題ありません。
6. **Deploy** をクリック。数十秒で公開され、
   `https://（プロジェクト名）.vercel.app` のようなURLが発行されます。

以降、GitHub Desktopで変更をpushするたびに、Vercelが自動的に再デプロイします。

## 5. 被験者に配布する4つのリンク

公開後のベースURLに `?group=1〜4` を付けたものが、そのままグループ別の
配布リンクになります。

| グループ | 言語 | フレーミング | 配布リンク |
|---|---|---|---|
| 1 | 日本語 | positive（ダウンロードすると20%残る） | `https://（プロジェクト名）.vercel.app/?group=1` |
| 2 | 日本語 | negative（ダウンロードしないと80%削除） | `https://（プロジェクト名）.vercel.app/?group=2` |
| 3 | 英語 | positive | `https://（プロジェクト名）.vercel.app/?group=3` |
| 4 | 英語 | negative | `https://（プロジェクト名）.vercel.app/?group=4` |

`?group=` を付けずにアクセスした場合は、4グループにランダムに割り振られます
（配布リンクを分けたくない場合はこちらでも利用できます）。

**送信前に必ず自分でそれぞれのリンクを開いてテストし、
スプレッドシートに1行ずつテストデータが入ることを確認してください。**
（確認後、テスト行は削除してから本番の配布を始めてください。）

## 6. 画面の流れ

1. **同意画面**：研究概要と、表示されるメッセージが模擬（フェイク）であることを明示し、
   チェックボックス＋ボタンで同意を取得。
2. **実験画面**：ページを開くと同時に、背景（ぼかし表示のアンケート）の上に
   カウントダウン付きの模擬警告モーダルが表示される。
   「ダウンロード」ボタン／「閉じる」ボタン／右上の✕のいずれかを押す
   （またはカウントダウンが0になる）とモーダルが消え、
   背景のアンケートが操作可能になる。
3. **アンケート**：英語レベル（CEFR）、3つの恐怖シナリオ（1〜5）、
   先ほどのメッセージへの対応行動、メッセージへの恐怖度（1〜5）を回答して送信。
4. **デブリーフィング画面**：実験の目的（言語×フレーミングによる詐欺メッセージ受容の違いの調査）
   を明示し、模擬メッセージであったことを再度伝えて終了。

## 7. スプレッドシートに記録される項目

| 列名 | 内容 |
|---|---|
| response_id | 回答ごとのランダムID |
| group | 1〜4（グループ番号） |
| lang | ja / en |
| framing | positive / negative |
| consent_time | 同意した時刻 |
| modal_shown_time | 模擬警告が表示された時刻 |
| modal_action | download / close / close_x / timeout（モーダルをどう処理したか） |
| modal_reaction_ms | 表示から処理までの反応時間（ミリ秒） |
| cefr / cefr_note | 自己申告の英語レベル／備考（試験スコア等） |
| fear_key / fear_stalker / fear_car | 3つの恐怖シナリオの評定（1〜5） |
| scam_action_choice | 模擬メッセージに対して選んだ行動 |
| scam_fear | 模擬メッセージへの恐怖度（1〜5） |
| user_agent | 使用ブラウザ情報 |
| submitted_at / server_received_at | 送信時刻（クライアント側／サーバー側） |

グループごとにフィルタ・ピボットテーブルをかければ、
4条件間の比較がそのままスプレッドシート上で行えます。

---

## 8. ご確認いただきたい点

- `config.js` 内の英語メッセージ（グループ3・4）は、いただいた原文の
  大文字小文字や文の切れ目を軽く整えています（意味・数値・フレーミングの方向は変更していません）。
  実験刺激として厳密な文言が重要な場合は、最終的な文面をご自身で必ずご確認ください。
- 同意画面の担当者名・連絡先は仮のプレースホルダーです。必ず実際の情報に置き換えてください。
- カウントダウンの秒数（既定 120秒）は `config.js` の `COUNTDOWN_SECONDS` で変更できます。
